I · COMMIT Gestionarea fișierelor și commit-urilor
01
Creat repository „Evaluare-Nr2" pe GitHub
Accesând platforma GitHub, am creat un repository nou cu numele Evaluare-Nr2, setat ca Public, cu un fișier README inițial generat automat.
github.com → New Public Add README
02
Adăugat fișierul index.ev.2.html în repository
Direct pe GitHub, prin interfața web, am creat un fișier nou cu conținut HTML de bază și l-am salvat cu un commit.
Add file → Create new file → Commit new file
03
Clonat repository-ul local
Am descărcat o copie locală a repository-ului de pe GitHub pentru a lucra cu el pe calculator.
Git Bash
$ git clone https://github.com/Nattalia903/Evaluare-Nr2.git
$ cd Evaluare-Nr2
✓ Cloning into 'Evaluare-Nr2'... done.
04
Creat local fișierul settings.xml
Am creat un fișier gol cu comanda touch, fără a-l adăuga încă în zona de staging.
Git Bash
$ touch settings.xml
05
Commit fișierul settings.xml
Am adăugat fișierul în staging area cu git add, apoi am salvat modificările în istoricul Git cu git commit.
Git Bash
$ git add settings.xml
$ git commit -m "Commit file settings.xml"
✓ 1 file changed, 0 insertions(+), 0 deletions(-)
06
Redenumit settings.xml → configuration.xml
Comanda git mv redenumește fișierul ȘI înregistrează automat această schimbare în staging area.
Git Bash
$ git mv settings.xml configuration.xml
07
Commit fișierul redenumit
Am salvat în Git operația de redenumire a fișierului din settings.xml în configuration.xml.
Git Bash
$ git add configuration.xml
$ git commit -m "Commit configuration.xml"
08
Modificat comentariul ultimului commit cu --amend
Comanda --amend permite editarea mesajului ultimului commit fără a crea unul nou. Util pentru corectarea greșelilor.
Git Bash
$ git commit --amend -m "Commit file configuration.xml"
[main 1911fbc] Commit file configuration.xml
09
Transferat toate modificările în repository remote
Comanda git push trimite commit-urile locale pe serverul GitHub, sincronizând repository-ul local cu cel remote.
Git Bash
$ git push origin main
79c4732..1911fbc main → main
10
Evaluare rezultat
Verificare finală a istoricului de commit-uri pentru a confirma că toate modificările sunt corecte.
Git Bash
$ git log --oneline
1911fbc (HEAD → main, origin/main) Commit file configuration.xml
1d088c1 Commit file settings.xml
79c4732 Create index.ev.2.html
0091f00 Initial commit
✓ 4 commit-uri înregistrate corect pe branch-ul main
II · COMMIT Revenire la un commit anterior
01
Adăugat remote fișierul page1.html
Am creat un fișier nou direct pe GitHub prin interfața web, simulând o modificare adusă de un coleg remote.
GitHub → Add file → Create new file → page1.html
02
Actualizat repository local cu git pull
Am descărcat modificările noi de pe GitHub (fișierul page1.html) în copia locală.
Git Bash
$ git pull origin main
✓ page1.html | 1 + · 1 file changed
03
Revenire la „Commit file configuration.xml" cu git reset
Comanda git reset --hard mută HEAD la un commit anterior, ștergând complet toate commit-urile și fișierele adăugate după acel punct.
Git Bash
$ git reset --hard 1911fbc
HEAD is now at 1911fbc Commit file configuration.xml
04
Actualizat repository remote cu force push
Deoarece istoricul local diferă de cel remote (am „rescris" istoricul), trebuie folosit --force pentru a forța actualizarea pe GitHub.
Git Bash
$ git push origin main --force
+ 1c14b9c...1911fbc main → main (forced update)
05
Evaluare rezultat
✓ page1.html eliminat · HEAD revenit la Commit file configuration.xml
I · BRANCH Creat branch stage și adăugat hosts.xml
01
Creat branch stage din main
Un branch este o ramură separată de dezvoltare. Comanda checkout -b creează și trece automat pe noul branch.
Git Bash
$ git checkout -b stage
Switched to a new branch 'stage'
$ git push origin stage
* [new branch] stage → stage
02-03
Creat și commit hosts.xml în branch stage
Am creat fișierul pe branch-ul stage și l-am salvat cu un commit specific acestui branch.
Git Bash
$ touch hosts.xml
$ git add hosts.xml
$ git commit -m "Commit file hosts.xml"
$ git push origin stage
04-05
Merge branch stage → main
Fuzionarea (merge) aduce toate modificările din stage în main. Fast-forward înseamnă că nu au existat conflicte.
Git Bash
$ git checkout main
$ git merge stage
Fast-forward · hosts.xml | 0 · 1 file changed
$ git push origin main
06
Evaluare rezultat
Git Bash
$ git log --oneline --all --graph
* 5dc3483 (HEAD → main, origin/stage, origin/main, stage) Commit file hosts.xml
* 1911fbc Commit file configuration.xml
✓ hosts.xml prezent în main și stage · merge reușit
II · BRANCH Creat branch dev și adăugat env.txt
01
Creat branch dev din stage
Branch-ul dev este creat pornind din stage, moștenind toate commit-urile acestuia.
Git Bash
$ git checkout stage
$ git checkout -b dev
Switched to a new branch 'dev'
02-03
Creat și commit env.txt
Fișierul env.txt a fost creat pe branch-ul dev și salvat cu comentariul cerut.
Git Bash
$ touch env.txt
$ git add env.txt
$ git commit -m "Commit file env.txt"
[dev 9167d3a] Commit file env.txt
04-05
Merge dev → stage și push
Modificările din dev sunt aduse în stage, apoi trimise pe GitHub.
Git Bash
$ git checkout stage
$ git merge dev
$ git push origin stage
bcad100..f2c2289 stage → stage
06
Evaluare rezultat
✓ env.txt prezent în stage · HEAD → stage, origin/stage
III · BRANCH Redenumit branch dev → develop
01
Redenumit local branch dev → develop
Comanda git branch -m redenumește un branch local fără a afecta remote-ul.
Git Bash
$ git branch -m dev develop
02
Actualizat repository remote
Am trimis branch-ul develop pe GitHub. Eroarea la ștergerea dev este normală — dev nu fusese trimis niciodată pe remote.
Git Bash
$ git push origin --delete dev
error: unable to delete 'dev': remote ref does not exist
# Normal — dev nu a existat pe remote
$ git push origin develop
* [new branch] develop → develop
03
Evaluare rezultat final
Git Bash
$ git branch -a
develop
main
* stage
remotes/origin/develop
remotes/origin/main
remotes/origin/stage
🎉 Toate task-urile completate · develop · main · stage active pe remote
